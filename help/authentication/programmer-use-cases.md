# 프로그래머 사용 사례 {#programmer-use-cases}

>[!NOTE]
>
>이 페이지의 컨텐츠는 정보용으로만 제공됩니다. 이 API를 사용하려면 Adobe의 현재 라이선스가 필요합니다. 무단 사용이 허용되지 않습니다.

## 개요 {#overview}

이 문서에서는 Adobe Primetime 인증에서 지원하는 프로그래머 통합 사용 사례를 요약합니다. 통합 프로젝트를 시작하기 전에 이 페이지에서 현재 지원되는 기능을 확인할 수 있습니다.

## 사용 사례 {#use-cases}


### 기본 통합: 단일 채널 네트워크에 대한 통합 인증 및 인증 {#basic-integration}

**우선순위** - 높음

**분류** - 경험 내에 호스팅된 1개의 채널 네트워크가 있는 단일 프로그래머 브랜드 TVE 앱

이렇게 하면 프로그래머가 MVPD에 대한 페더레이션 자격 검사를 사용하여 자신의 브랜드 TVE 앱*에서 프리미엄 콘텐츠를 제공할 수 있습니다. requestorID는 컨텐츠를 뷰어에 제공하는 애플리케이션의 브랜드와 일치하도록 맞아야 합니다. 이 시나리오에는 Adobe Primetime 인증 요청자 ID와 권한 부여에 대해 확인되는 리소스 ID 간에 1~1 관계가 있습니다.

>[!NOTE]
>
>TVE 앱은 이 문서에서 다양한 유형의 애플리케이션(웹 앱, 모바일 앱 등)을 전체적으로 참조하는 데 사용됩니다. Adobe Primetime 인증에서 지원합니다. 아래 플랫폼 열에는 특정 사용 사례에 대해 지원되는 플랫폼에 대한 세부 사항이 포함될 수 있습니다.

#### 특정 사용 사례(대부분의 통합에 공통됨) {#sp-use-cases-basic-int}

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|:--------:|:-----------------------------------------------------:|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------:|:----------------------------------------------------------------------------:|:-----------------------------------------:|
| 높음 | 프로그래머 TVE 앱의 MVPD 검색 | 사용자가 프로그래머의 브랜드 TVE 앱에서 시작하며 MVPD 공급자를 선택하라는 메시지가 표시됩니다. | 웹(SWF/JS) 모바일(iOS/Android) 클라이언트 없는 API(두 번째 화면용) |  |
| 높음 | 프로그래머 TVE 앱의 페더레이션 인증 | 사용자가 프로그래머의 브랜드 TVE 앱에서 시작되고 MVPD 공급자를 선택하면 사용자는 MVPD의 자체 로그인 페이지로 전환되어 자격 증명을 입력합니다. | 웹(SWF/JS) 모바일(iOS/Android) |  |
| 높음 | 프로그래머 TVE 앱에서 인증 | 사용자가 인증되면 Programmer의 TVE 앱은 MVPD에게 사용자의 자격을 확인하기 위해 백채널 인증 요청을 수행할 수 있습니다. 일반적으로 이는 채널 네트워크가 사용자 MVPD 구독 패키지에 있는지 확인하는 것입니다.                                  이 경우 요청자 ID 및 리소스 ID는 최대 1:1과 일치합니다. | 모든 플랫폼 |  |
| Medium | 프로그래머 TVE 앱에서 로그아웃 | 사용자가 Adobe Primetime 인증 AuthN/AuthZ 토큰을 로그아웃하고 지울 수 있습니다. 많은 경우, 이렇게 하면 MVPD에서도 사용자를 로그아웃합니다. 그러나 MVPD는 이 지원 여부에 따라 다릅니다. 항상 Adobe Primetime 인증 세션 및 토큰을 지웁니다. | XBox Native를 제외한 모든 플랫폼 | 여러 MVPD가 이 기능을 지원하지 않습니다. |
| 높음 | 사이트 및 앱에서 단일 사인온 | 사용자가 다시 로그인하지 않고 사이트 및 앱에서 로그인 세션을 공유할 수 있도록 합니다. | Clientless API를 제외한 모든 플랫폼 | 일부 MVPD에는 SDK 1.7 이상이 필요합니다. |

### 여러 채널 네트워크를 호스팅하는 단일 TVE 앱 {#single-app-multi-channel}

**우선순위**- 높음

프로그래머가 뷰어의 동일한 브랜드 대상에서 여러 컨텐츠 채널 네트워크를 집계할 수 있도록 합니다.

#### 특정 사용 사례 {#sp-use-cases-singl-tve-app}

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 높음 | 고유 채널 인증 | 사용자는 동일한 TVE 앱 내에서 여러 채널 네트워크의 콘텐츠를 볼 수 있습니다. 프로그래머는 각 채널 네트워크에 고유한 인증 호출을 수행하여 사용자의 자격 증명을 확인할 수 있습니다. | 모든 플랫폼 | 이제 모든 MVPD가 이 기능을 일부 형식으로 지원합니다. |
| 낮음 | 프리플라이트 인증 쿼리 | 이렇게 하면 프로그래머가 단일 API 호출에서 사용자가 패키지에 있는 채널을 확인할 수 있습니다. 이 작업은 실제 AuthZ가 사용자가 액세스할 수 없는 사용자 인터페이스에서 컨텐츠를 필터링하도록 호출하기 전에 수행됩니다. |  | 대부분의 MVPD는 이 데이터를 아직 사용자 속성으로 표시하지 않으므로 Adobe은 실제로 AuthZ 호출을 수행하여 이 데이터를 가져옵니다. 또한 대부분의 MVPD는 한 번의 호출로 여러 채널을 지원하지 않으므로 한 번에 5개로 제한됩니다.                             프로그래머가 프리플라이트 검사를 위해 필요한 채널의 수를 확인하는 것은 매우 중요합니다. 번호가 어떻게 되든지 간에, 우리는 MVPD에 문제가 없는지 확인해야 합니다. 대부분의 MVPD는 현재(2013년 3분기) 5개 이상의 채널을 지원하지 않습니다. |

### 자산 수준 인증 {#asset-level-authz}

**우선순위** - 낮음

**분류** - 권한 부여 요청 시 자산 식별자 전달

**플랫폼** - 모든 플랫폼

#### 특정 사용 사례 {#sp-use-cases-asset-lvl-authz}

MVPD가 각 AuthZ 호출에서 자산 수준 분석을 가져올 수 있도록 합니다. 이를 통해 Adobe Primetime 인증 AuthZ 캐시를 무효화할 수 없습니다.

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|-------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------|-------------|--------------------------------------|
| 낮음 | 인증 요청 시 자산 식별자 전달 | MVPD가 각 AuthZ 호출에서 자산 수준 분석을 가져올 수 있도록 합니다.  Adobe Primetime 인증 AuthZ 캐시를 무효화하는 단점이 있습니다. | 모든 플랫폼 | 현재 MVPD는 하나만 지원합니다. |




### 자녀 보호 {#parental-controls}

**우선순위** - 낮음

Programmer의 TVE 앱에 MVPD 사용자 계정 제한을 적용할 수 있도록 합니다.

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|-----------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------|-----------------------------------|
| 낮음 | 사용자 속성에 따라 컨텐츠 필터링 | 사용 가능한 콘텐츠 목록을 사용자에게 렌더링하기 전에 프로그래머가 사용자에 대해 허용되는 최대 등급을 확인할 수 있도록 합니다. | 웹(Flash/JS) 모바일(iOS/Android) | 현재 하나의 MVPD에서만 작동합니다. |
| 낮음 | AuthZ 요청에 컨텐츠 등급 전달 | 등급은 일반적으로 자산 수준에서 지정되므로 Programmer가 사용자가 AuthZ 요청의 일부로 보고 싶은 컨텐츠의 특정 등급을 MVPD와 #3로 전달할 수 있습니다. | 모든 플랫폼 | 현재 하나의 MVPD에서만 작동합니다. |

#### 프로그래머 브랜드당 MVPD 통합 사용자 지정 {#mvpd-int-cust-prog-brand}

**우선순위** - 중간

AuthN 또는 AuthZ 오류 메시지에 대해 사용자 지정 경험을 활성화합니다.

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------|-----------------------------------------|
| Medium | AuthN 요청에 서비스 공급자 식별자를 전달합니다. | 서비스 공급자에 대한 MVPD 로그인 페이지에서 특정 브랜딩을 사용하도록 설정합니다. 또한 Univision용 스페인어 등의 대상자와 일치하도록 기본값을 자동으로 선택할 수도 있습니다. | 모든 플랫폼 | MVPD에 따라 다릅니다. 일부는 이를 지원하지 않습니다. |
| Medium | AuthZ 응답에 대한 사용자 지정 오류 메시지 | 패키지를 업그레이드하는 링크로 업셀할 특정 메시지를 포함할 수 있는 MVPD의 프로그래머 또는 브랜드별 오류 메시지를 사용하도록 설정합니다. | 웹, Android, iOS | MVPD에 따라 다릅니다. 일부는 이를 지원하지 않습니다. |


### 연결된 장치 사용 사례 {#connected-devices}

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|-------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------|---------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Medium | 앱 및 콘솔 간 XBox LiveID SSO | 사용자가 자신의 LiveID 계정에 연결된 앱과 다른 게임 콘솔 간에 AuthN 세션을 공유할 수 있도록 합니다. | XBox 기본 SDK | 대부분의 MVPD는 일반적인 모델이 토큰이 사용자가 아니라 장치에 바인딩되므로 이를 좋아하지 않습니다.                             가능하면 이 방법을 더 이상 권장하지 않습니다. |
| 높음 | 장치에서 appID에 연결된 토큰이 있는 장치 | 프로그래머가 토큰의 MVPD 자격 부여를 발급된 장치의 appID에 바인딩할 수 있도록 합니다. | Clientless API | 이렇게 하면 연결된 장치가 토큰에 대한 표준 전달 구현에 더 가깝게 정렬됩니다.                             여전히 장치 전체 ID로 개선해야 합니다. |

### 장치별 AuthN TTL 길이 {#authn-ttl-length}

일반 채널과 같이 MVPD 자격 데이터베이스에 있는 리소스가 아닐 수 있는 특수 이벤트에 대해 TVE 자격 부여 기능을 활성화합니다.

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------|--------------------------------------------------------------------------------------------------------------------------|
| 높음 | 플랫폼당 다른 TTL 값 설정 | 프로그래머가 웹, 모바일 및 연결된 장치에 대해 다른 TTL 길이를 설정할 수 있도록 합니다. 현재 Adobe Primetime 인증에서는 3개의 개별 TTL 값을 갖는 기능을 지원합니다. 웹(Flash) 모바일/HTML5 클라이언트 없는 - 연결된 장치 |  | 일부 MVPD는 TTL을 동적으로 설정합니다. 필요한 경우 구성 설정을 사용하여 Adobe이 이러한 동적 설정을 재정의할 수 있습니다. |

### 특별 이벤트 기반 애플리케이션 {#special-event}

**우선순위** - 낮음

일반 채널과 같이 MVPD 자격 데이터베이스에 있는 리소스가 아닐 수 있는 특수 이벤트에 대해 TVE 자격 부여 기능을 활성화합니다.

| 우선순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 |
|--------|---------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------|------------------------------------------|
| 낮음 | 여러 채널을 이벤트의 프록시로 사용 | 이는 올림픽을 위해 수행됐으며 가입자는 패키지에 있는 두 개의 다른 채널을 접속해야 한다. 이 경우 Adobe Primetime 인증에서는 새 resourceID를 만들고 모든 MVPD가 끝에 특정 채널에 대한 매핑을 수행하도록 합니다.  그것은 충분한 사전 공지로써 잘 되었다. 대부분의 MVPD가 여러 리소스 호출을 지원하지 않으므로 이 작업이 중요합니다. | 모든 플랫폼 | 적절한 알림과 함께 모든 MVPD에서 지원됩니다. |
| 낮음 | 기존 채널 리소스를 사용하는 새로운 특별 이벤트 애플리케이션 | 이건 3월 광기에 대한 겁니다 콘텐츠 공급자가 새 requestorID를 사용하여 새 앱을 만들었습니다. 시스템에서 새 requestorID에 대한 지원을 추가하는 데 모든 MVPD가 필요합니다. resourceID는 일반적인 채널이었습니다.  또한 일부 MVPD는 새 요청자 아래에 유효한 채널을 매핑해야 하므로 이러한 경우에 더 많은 시간이 필요합니다. | 모든 플랫폼 | 적절한 알림과 함께 모든 MVPD에서 지원됩니다. |
| 낮음 | 기존 요청자 ID, resourceID | 이번 대회는 마스터스 골프 주말 토너먼트에서 열렸습니다. 그것은 단지 이틀 동안 작은 행사였고, 마스터즈는 컨텐츠를 표시하기 위해 그들의 모바일 앱을 가지고 있었습니다. 프로그래머는 Adobe Primetime 인증 트래픽을 위해 비용을 지불하고 표준 요청자 ID 및 resourceID만 사용할 계획이었습니다. 유일한 트릭은 Programmer가 Master와 요청자ID 서명을 위해 모바일 인증서를 공유하도록 하고, 해당 주말의 백업 인증서로 해당 구성에 추가하도록 하는 것입니다. | 모든 플랫폼 | MVPD에 영향을 주지 않음 |

### 콘텐츠 서버 통합 {#content-server-integration}

**우선순위**- 중간

비디오 스트림을 클라이언트 플레이어로 릴리스하기 전에 미디어 토큰 유효성 검사를 활성화합니다.
| 우선 순위 | 사용 사례 | 설명 | 플랫폼 | MVPD 노트 | |—|—|—|—|—| | 높음 | 프로그래머 Federated Player - 페이지 수준 인증 사용 | Adobe Primetime 인증 API는 페이지의 JavaScript에서 수행되고 토큰이 플레이어에 전달됩니다. 토큰은 다음 두 가지 방법으로 유효성 검사 서비스에 전달할 수 있습니다. 스트림 URL 외부 인터페이스 API FlashVars의 쿼리 문자열에서 전달된 유효성 검사 서비스 URL 매개 변수의 매개 변수를 가져옵니다 | | | | 미디어 | 프로그래머 Federated Player - 내부 플레이어 인증 사용 | Adobe Primetime 인증 API는 플레이어 SWF의 ActionScript에서 수행되므로 콜백에서 플레이어에서 토큰을 사용할 수 있습니다.                                                                                                                                                                                         | | | | 높음 | 신디케이트된 플레이어 - iFrame을 사용하여 플레이어를 래핑하는 페이지 수준 인증이 있는 MVPD 포털에서 호스팅 | 페이지 수준 인증이 있는 플레이어와 유사하지만 플레이어 페이지 래퍼가 MVPD 포털에 프레임이 있습니다. 인증은 MVPD 포털에서 별도로 수행해야 합니다.                                                                                                                                                    |           |                        |
