# MVPD 통합 기능

>[!NOTE]
>
>이 페이지의 컨텐츠는 정보용으로만 제공됩니다. 이 API를 사용하려면 Adobe의 현재 라이선스가 필요합니다. 무단 사용이 허용되지 않습니다.

## 개요 {#mvpd-int-features-overview}

Adobe Primetime 인증은 TV Everywhere의 최신 표준을 지원합니다. API는 **CableLabs OLCA(온라인 컨텐츠 액세스) 사양**- 온라인 소스에서 유료 TV 고객에게 비디오를 제공하기 위한 기술 요구 사항 및 아키텍처를 제공합니다. Adobe은 2011년 6월에 공동 CableLabs 옵트 테스트 프로젝트에 참여했으며 서비스 공급자 구현을 위한 테스트 프로세스를 통과했습니다.  Adobe은 또한 **OATC(Open Authentication Technical Consortium)** 그리고 이 기구의 일부로서 몇 개의 소위원회의 사양 초안 작성 프로젝트에 참여하고 있습니다.

Primetime 인증 OLCA 규정 준수와 OATC에 대한 Adobe의 참여를 설명했으므로, Primetime 인증은 실제로 &quot;프로토콜에 관계없이&quot; 인증을 수행하는 것이 중요합니다.  그러나 TVE 시대의 이 단계에서 Primetime 인증은 OLCA 표준을 지향합니다.  이 표준에서는 TVE 기능을 구현하는 여러 TVE 플레이어(프로그래머, MVPD 및 서비스 공급자)에 대해 현재 합의된 방법을 설명합니다. 이러한 기능 중 다수는 기능 구현 방법에 대한 세부 사항 및 예를 제공하는 관련 페이지에 대한 링크와 함께 아래 표에 나열되어 있습니다.

표의 정보는 모든 MVPD 통합에서 일관된 기능을 위해 Primetime 인증 통합 프로세스를 구동하기 위한 것입니다. 우선 순위 열은 기능을 A, B 및 C로 분류합니다.

* A - 통합에서 초기 롤아웃에 대한 &quot;필수&quot; 기능입니다.
* B - 초기 롤아웃 후 추가할 초기 통합에 대한 중요한 개선 사항입니다.
* C - 이러한 기능은 &#39;B&#39; 요구 사항 후에 구현할 수 있는 통합에 대한 추가적인 개선 사항입니다.

| 1.12 | **인증 응답의 사용자 메타데이터** | MVPD AuthN 응답에는 자격 흐름 중 사용자 경험의 개인화를 위한 힌트 역할을 하는 사용자 메타데이터가 포함될 수 있습니다. 이 요구 사항을 통해 MVPD에서 프로그래머로의 자녀 보호 힌트가 활성화됩니다. | B- |  |
| 1.13 | MVPD가 시작한 인증 | 뷰어는 MVPD 포털에서 성공적인 AuthN 세션을 완료한 다음 프로그래머의 TVE 사이트로 이동합니다. 사용자에게 MVPD 선택기를 묻는 메시지가 표시되지 않으며 자동으로 인증됩니다. | B- |  |
| 1.14 | UserID 범위 지정 | MVPD UserID는 두 가지 형식으로 발생해야 합니다. 하나는 프로그래머로 범위가 지정되고 다른 하나는 사기 범위가 지정된 Adobe 범위입니다.  이를 통해 Adobe은 추가적인 암호화/난독화 없이 프로그래머 범위 MVPD UserID를 공유할 수 있습니다. | C |  |
| 1.15 | 자산 기반 인증 | AuthN이 완료되면 필요에 따라 네트워크, 표시, 자산, 자녀 보호 등급 등을 포함할 수 있는 구조화된 데이터를 전달하여 백그라운드에서 AuthZ가 발생할 수 있습니다. 이를 통해 프로그래머에서 MVPD로 전달되는 모든 AuthZ 호출에 대해 자녀 보호를 할 수 있습니다. | C |  |
| 1.16 | MVPD가 시작한 로그아웃 | 뷰어에는 Programmer의 사이트 또는 앱과 MVPD가 있는 인증된 세션이 있습니다. 뷰어는 MVPD 사이트에서 페더레이션 로그아웃을 시작하여 모든 페더레이션 프로그래머 사이트의 세션도 지울 수 있습니다. | C |  |
| 1.17 | 권한 부여 의무 | MVPD는 로깅 또는 OLCA(새로 고친 최대 보호자 통제 등급)와 같은 AuthZ 응답에 추가 조건을 제공합니다. | C |  |
| 1.18 | IP 주소 컨텍스트 | MVPD가 IP 주소를 명시적으로 전달해야 합니다. AuthZ의 경우 호출이 서버측인 경우 AuthZ 호출에서 사용자가 오는 위치에 대한 MVPD 사기 추적 정보를 제공합니다. | C |  |
| 1.19 | 동적으로 정의된 토큰 지속성 TTL(Time-to-Live) 값 | MVPD는 응답의 속성을 통해 Primetime 인증 토큰의 TTL을 동적으로 설정할 수 있으므로 Adobe이 TTL 변경에 대한 루프를 벗어납니다. | C- |  |
| 1.20 | 장치 유형 | MVPD는 AuthN 또는 AuthZ 요청에 장치 유형 전달을 지원합니다. 이 속성은 MVPD에 컨텐츠가 사용되는 장치의 특성을 알려, AuthN 또는 AuthZ 토큰의 TTL을 조정하여 장치에 대한 자체 보안 고려 사항을 준수하도록 합니다. | C- | 이 메서드는 Primetime 인증 측에서 모든 장치 유형을 구성 설정으로 노출하기가 어려울 수 있는 클라이언트 없는 플랫폼에 유용합니다. |
