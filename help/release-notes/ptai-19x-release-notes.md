### 버전 19.7.1

**시기: 7** 월 10일 수요일

* 사이:라이브 스트림의 EXT-X-CUE-OUT 광고 브레이크 신호를 지원하는 형식 값을 위한 EXT-X-ASSET 태그의 특성에서 데이터를 전달하는 일반 매크로를 추가했습니다. 예:#EXT-X-CUE-OUT 태그와 함께 사용되는 태그:#EXT-X-ASSET:CAID=75BCD15,GENRE=News,Program=NewsAt10 매크로:#을 사용하여 뉴스(GENRE 속성)를 광고 호출 URL #로 전달하여 NewsAt10(프로그램 속성)을 광고 호출 URL 예외에 전달할 수 있습니다.이전 버전과의 호환성을 위해 # 및 # 기능도 동일합니다. 두 매크로 모두 CAID 특성의 값을 16진수로 변환한 후 전달할 수 있습니다. 위의 예에서 긴 값은 123456789이고 16진수 값은 75BCD15입니다. 두 매크로 모두 123456789를 광고 호출 URL로 전달하는 데 사용됩니다. 매크로는 항상 #로 시작합니다. 매크로는 대/소문자를 구분하지만 EXT-X-ASSET 태그의 특성은 그렇지 않습니다. 즉, EXT-X-ASSET 태그에서는 프로그램과 프로그램이 모두 허용됩니다

   * 슬라이딩 창(라이브 재생 목록) 길이 4분

* 보안 업데이트

### 버전 19.4.1

**시기:** 수요일, 4월 10일 오전 2시 30분 동부 표준시, 4월 10일 오전 4시 30분 동부 시간

* CRS:CRS 재패키징 API는 더 이상 HTTP POST 명령을 지원하지 않습니다. CRS Repackaging API는 자동으로 (301) HTTP POST 명령을 HTTPS로 리디렉션합니다
   * 5월 20일부터 HTTP POST 명령에 대한 HTTP->HTTPS 리디렉션이 해제됩니다.
   * CRS Repackaging API를 사용하여 광고를 미리 패키지하는 경우 5월 20일까지 POST 명령을 HTTPS로 전환하십시오
* CRS:고객의 CDN 기원에 CRS 자산을 업로드하기 위한 아키텍처 및 워크플로우 재설계
   * CDN 오리건당 작업 프로세스는 구분되므로 하나의 CDN 원본에 대한 업로드 병목 현상은 다른 CDN 원본에 대한 업로드에 영향을 주지 않습니다
   * 기타 이점:고객의 CDN 기원에 대한 CRS 작업 처리 시간 및 업로드 속도 개선
* 사이:사이드카 JSON v2 형식에 비디오 광고에 대한 ClickThrough 및 ClickTracking URL을 추가했습니다.
   * 새 JSON 배열 속성 &quot;videoClicks&quot;가 &quot;trackingURL&quot; 속성 다음에 나옵니다
   * &quot;event&quot; 값 이름은 &quot;clickThrough&quot; 및 &quot;clickTracking&quot;이 되며 startTime 값이 없습니다
* 사이:CRS 자산에 대해 삽입될 때마다 CRS 자산의 조회 레코드 만료를 30일까지 연장하는 기능을 추가했습니다.
   * 이전 동작:CRS 자산 조회 레코드는 각 창의 메모리 캐시에 저장됩니다. CRS 자산 조회 레코드는 메모리 캐시에 추가된 후 30일 후에 자동으로 제거됩니다. 크리에이티브의 CRS 자산 조회 레코드를 메모리 캐시에서 제거한 후 창에서 다시 채우려면 해당 창에서 해당 크리에이티브가 3번 발견되어야 합니다
   * 새로운 동작:CRS 자산을 삽입하기 위해 창이 CRS 자산 조회 레코드에 액세스하는 경우 해당 창에서 CRS 자산 조회 레코드의 만료일이 30일까지 연장됩니다. 따라서 자주 사용되는 CRS 에셋은 가장 최근에 사용한 후 30일이 경과할 때까지 창의 메모리 캐시에서 제거되지 않습니다
   * 새로운 동작은 시스템 전체에 적용되며 성능 저하가 감지되면 전환될 수 있습니다
* 사이:라이브 스트림에만 대한 WebVTT 매니페스트 조작 비헤이비어 업데이트
   * 이전 동작:WebVTT 매니페스트에서만 삽입된 광고 앞과 삽입된 광고 나누기의 마지막 세그먼트 뒤에 삽입되는 EXT-X-DIS연속성 태그를 제거합니다
   * 새로운 동작:SSAI 부트스트랩 URL에 허용된 값이 true 및 false인 새 매개 변수 vttdisc를 추가했습니다.
      * vttdisc=true:EXT-X-DISCONTINUITY 태그는 삽입된 광고 앞과 삽입된 광고 중단의 마지막 세그먼트 뒤에 삽입되며 오디오/비디오 및 오디오 전용 매니페스트에 대한 동작과 일치합니다
      * vttdisc=false(이전 동작과 동일):WebVTT 매니페스트에서만 삽입된 광고 앞과 삽입된 광고 나누기의 마지막 세그먼트 뒤에 삽입되는 EXT-X-DIS연속성 태그를 제거합니다
      * vttdisc 매개 변수가 생략되거나 true/false가 아닌 값이 있으면 vttdisc가 기본값이 true로 설정됩니다
* 사이:보안 업데이트 및 소프트웨어 버전 업데이트
   * Java:TLS 1.2(HTTPS)를 통해 실행되는 광고 호출에 대한 추가 암호 세트를 지원하도록 Java 버전 업데이트

   * 이전 조회 키 구조:영역, 광고 시스템, 광고 ID, 크리에이티브 ID, 크리에이티브 URL 및 형식 매개 변수(대상 기간, 출력 형식, 대상 CDN)
   * 기존 CRS 에셋에 대한 조회 키는 프로덕션 릴리스 전의 새 구조와 일치하도록 업데이트되지만 조회 키 업데이트와 프로덕션 릴리스 간에 트랜스코딩된 새 에셋이 누락될 수 있습니다. 이 경우 릴리스 이후 다음에 만나면 새로운 CRS 요청을 시작합니다

* CRS:특정 광고 시스템, 광고 ID, 크리에이티브 ID, 크리에이티브 URL 및/또는 크리에이티브 포맷의 차단 목록/허용 목록 CRS 요청 기능 추가

   >참고
   >
   >동일한 광고에 대한 동적 값(예: URL의 동적 매개 변수)이 있는 광고 공급자가 검색되면 Adobe이 차단 목록 규칙을 추가합니다. 동적 구성 요소가 제공자에 의해 해결되거나 표준화 규칙을 통해 해결되면 이러한 차단 목록 규칙이 비활성화됩니다.

   * 영역에 대한 차단 목록 또는 허용 목록 규칙을 추가하려면 기술 계정 관리자에게 문의하십시오.
