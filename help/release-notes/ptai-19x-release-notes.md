### 버전 19.7.1

**시기:** 7월 10일 수요일

* 사이:특정 고객에 대한 구성 변경 사항:
   * 슬라이딩 창(라이브 재생 목록) 길이 4분
   * 매니페스트 서버가 소스 컨텐츠를 가져올 때 소켓 시간 초과 예외가 발생하는 경우 매니페스트 서버는 500 대신 HTTP 응답 코드(404)를 반환합니다.
* 보안 업데이트

### 버전 19.4.1

**시기:** 4월 10일 수요일 오전 2:30 동부 표준시 4월 10일 수요일 오전 4:30 동부 시간

* CRS:CRS 재패키징 API는 더 이상 HTTP POST 명령을 지원하지 않습니다. CRS Repackaging API는 자동으로 (301) HTTP POST 명령을 HTTPS로 리디렉션합니다.
   * 5월 20일부터 HTTP->HTTP POST 명령에 대한 HTTPS 리디렉션이 해제됩니다.
   * CRS Repackaging API를 사용하여 광고를 미리 패키지하는 경우 5월 20일까지 POST 명령을 HTTPS로 전환하십시오
* CRS:고객의 CDN 원본에 CRS 에셋을 업로드하기 위한 아키텍처 및 워크플로우 재설계
   * CDN 출처당 작업 프로세스는 분리되므로 한 CDN 원본에 대한 업로드 병목 현상은 다른 CDN 원본에 대한 업로드에 영향을 주지 않습니다
   * 기타 이점:CRS 작업 처리 시간 및 고객의 CDN 원본에 대한 업로드 비율이 개선되었습니다.
* 사이:사이드카 JSON v2 형식에 비디오 광고에 대한 ClickThrough 및 ClickTracking URL이 추가되었습니다.
   * 새 JSON 배열 속성 &quot;videoClicks&quot;는 &quot;trackingURL&quot; 속성 다음에 옵니다.
   * &quot;event&quot; 값 이름은 &quot;clickThrough&quot; 및 &quot;clickTracking&quot;이며 startTime 값이 없습니다.
* 사이:CRS 자산의 경우 CRS 자산이 삽입될 때마다 30일까지 CRS 자산의 조회 레코드 만료를 확장하는 기능이 추가되었습니다.
   * 이전 동작:CRS 자산 조회 레코드는 각 창의 메모리 캐시에 저장됩니다. CRS 자산 조회 레코드는 메모리 캐시에 추가된 후 30일 후 자동으로 제거됩니다. Creative의 CRS 자산 조회 레코드를 멤버 캐시에서 제거한 후 창에서 다시 채우려면 해당 창에서 해당 크리에이티브가 3번 발견되어야 합니다.
   * 새로운 동작:CRS 자산을 삽입하기 위해 창이 CRS 자산 조회 레코드에 액세스하면 CRS 자산 조회 레코드의 만기가 해당 창에서 30일까지 연장됩니다. 따라서 자주 사용되는 CRS 에셋은 가장 최근에 사용한 후 30일이 경과할 때까지 창의 메모리 캐시에서 제거되지 않습니다
   * 새로운 동작은 시스템 전체에 적용되며 성능 저하가 감지되면 전환될 수 있습니다
* 사이:라이브 스트림에 대해서만 WebVTT 매니페스트 조작 비헤이비어가 업데이트되었습니다.
   * 이전 동작:WebVTT 매니페스트에서만 삽입된 광고 앞 및 삽입된 광고 분리의 마지막 세그먼트 뒤에 삽입할 EXT-X-DISCONTINUITY 태그를 제거합니다
   * 새로운 동작:새 매개 변수 vttdisc를 true와 false의 허용된 값과 함께 SSSAI 부트스트랩 URL에 추가했습니다.
      * vttdisc=true:EXT-X-DISCONTINUITY 태그는 삽입된 광고 앞 및 삽입된 광고 중단의 마지막 세그먼트 뒤에 삽입되며 오디오/비디오 및 오디오 전용 매니페스트의 동작과 일치합니다
      * vttdisc=false(이전 동작과 동일):WebVTT 매니페스트에서만 삽입된 광고 앞 및 삽입된 광고 분리의 마지막 세그먼트 뒤에 삽입할 EXT-X-DISCONTINUITY 태그를 제거합니다
      * vttdisc 매개 변수가 생략되거나 true/false가 아닌 값이 있으면 vttdisc가 기본적으로 true로 설정됩니다
* 사이:보안 업데이트 및 소프트웨어 버전 업데이트
   * Java:TLS 1.2(HTTPS)를 통해 실행되는 광고 호출에 대한 추가 암호화 세트를 지원하도록 Java 버전을 업데이트했습니다.
