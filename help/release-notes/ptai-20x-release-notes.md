# Primetime Ad Insertion 20.12.1 릴리스 노트

Primetime Ad Insertion 20.12.1 릴리스 노트는 2020년 Primetime Ad Insertion에서 새로운 기능 또는 변경된 사항, 해결된 문제 및 알려진 문제를 설명합니다.

## PTAI 20.12.1의 새로운 기능

**시기:** 2020년 12월 8일 화요일 오전 1시부터 오전 4시까지 동부 시간

**변경 사항**

* 2020년 11월 30일에 발생한 Primetime Ad Insertion의 간헐적인 클라이언트 연결(5xx) 문제를 해결하기 위한 핫픽스를 포함합니다.

## 이전 릴리스 버전의 개선 사항 및 수정 사항

### 버전 20.11.1

**시기:** 2020년 11월 5일 목요일 오전 2시부터 오전 5시까지 동부 시간

**변경 사항**

* 유지 보수 업데이트.

### 버전 20.10.2

**시기:** 2020년 10월 29일 목요일 오전 12:01~06:00 동부 표준시

**변경 사항**

* 유지 보수 업데이트.

### 버전 20.10.1

**시기:** 2020년 10월 13일 화요일 오전 3시부터 오전 7시까지 동부 시간

**변경 사항**

* 유지 보수 업데이트.

### 버전 20.9.3

**시기:** 2020년 9월 30일 수요일 오전 3시 30분 ~ 오전 6시 30분

**변경 사항**

* Bootstrap API 매개 변수 추가됨 `ptparallelstream`. 이렇게 하면 CMAF 중복 제거된 오디오 또는 비디오 스트림을 병렬로 요청하는 플레이어를 사용하는 고객이 오디오 및 비디오 트랙의 광고가 일관되도록 할 수 있습니다. 이 기능을 활성화하려면 매개 변수 값을 true로 설정하고, 비활성화하려면 생략합니다.

### 버전 20.9.2

**시기:** 2020년 9월 15일 화요일 오전 3시 30분부터 오전 6시 30분까지 동부 시간

**개선 사항**

* 를 사용하여 비선형 광고 유형을 포함할 수 있도록 지원합니다. `EXT-X-MARKER` 태그 사이에 코드를 삽입하지 마십시오.
자세한 내용을 보거나 이 기능을 활성화하려면 기술 지원 담당자에게 문의하십시오.

* 공급자가 응답하는 데 너무 오래 걸리는 경우 전체 광고 해결 시간을 제한하는 데 대한 지원이 제공되었습니다. 제한을 활성화하려면 부트스트랩 API 매개 변수를 설정합니다 `ptadtimeout` 을 밀리초 단위의 값으로 바꿉니다.

   >[!NOTE]
   >
   >이 시간 제한은 광고 크리에이티브 요청이 아닌 광고 요청에만 적용됩니다.

### 버전 20.9.1

**시기:** 2020년 9월 1일 화요일 오전 3시 30분부터 오전 7시 30분까지 동부 시간

**변경 사항**

* HLS/CMAF를 사용하는 고객의 경우 EXT-X-MAP에 CDN 토큰이 누락되거나 EXT-X-MAP 태그가 DVR 창에서 잘못 롤아웃되는 문제가 해결되었습니다.

### 버전 20.8.4

**시기:** 2020년 8월 19일 수요일, 오전 3시 30분 ~ 오전 7시 30분

유지 보수 업데이트.

### 버전 20.8.1

**시기:** 2020년 8월 4일 화요일 오전 3시부터 오전 6시까지 동부 시간

**개선 사항 및 수정 사항**

유지 보수 업데이트.

### 버전 20.7.1

**시기:** 2020년 7월 9일 목요일 오전 3시 ~ 오전 5시(동부 표준시)

**새로운 기능 및 향상된 기능**

* 큐를 식별하기 위해 공급자 광고 시작/종료 메시지 또는 브레이크 시작/종료 메시지를 사용하는 SCTE35가 개선되었습니다.

* X-ADBE-AI-X1 헤더가 문제 해결에 도움이 되는 추가 정보로 업데이트되었습니다.

* 향상된 지표 집계입니다.

* 세션 상태 패널에 대한 향상된 SSAI 콘솔 대시보드

### 버전 20.6.2

**시기:** 2020년 6월 18일 목요일, 오전 3시 ~ 오전 4시(동부 표준시)

**개선 사항**

밀리초 단위의 정밀도가 필요한 비디오 클라이언트에 대한 스트림 동기화가 개선되었습니다. 다음에 대한 밀리초 정밀도를 활성화하려면 Adobe 지원 센터에 문의하십시오. `#EXT-X-PROGRAM-DATE-TIME tags`.

### 버전 20.6.1

**시기:** 2020년 6월 2일 화요일 오전 3시부터 오전 5시까지 동부 시간

**새로운 기능**

서버측 구성을 통해 다음과 같은 새로운 기능을 활성화하려면 Adobe 지원 센터에 문의하십시오.

* 매니페스트 조작: 이제 HTTP와 HTTPS 간에 HLS 세그먼트 및 리소스 URL을 변환하여 백엔드 요청에 대한 TLS 핸드셰이크를 줄여 성능을 높일 수 있습니다. 광고/컨텐츠 조각을 동일한 CDN에 통합하는 데 사용할 수도 있습니다.

* 긴 양식 VOD: 긴 양식 VOD 자산으로 세션 유지를 유지 관리하기 위해 API가 개선되었습니다.

**버그 수정**

* WebVTT 조각이 요청된 원본 프로토콜과 관계없이 항상 http 프로토콜에서 요청되던 문제를 해결했습니다.

* 광고에서 컨텐츠로 다시 전환할 때 재생 목록의 맨 위에서 EXT-X-DISCONTINUITY 태그가 제거되던 문제를 수정했습니다. 이 수정 사항을 활성화하려면 Adobe 지원 센터에 문의하십시오.

### 버전 20.5.1

**시기:** 2020년 5월 5일 화요일 오전 4:00 - 오전 5:00 동부 표준시

* If-Modified-Since 헤더가 전송될 때 올바른 CORS 헤더가 제공되도록 하는 문제를 해결했습니다.

* CRS 대시보드에 대한 버그가 수정되었습니다.

* 유지 보수 업데이트.

### 버전 20.3.4

**시기:** 2020년 4월 1일 수요일, 오전 3시부터 오전 4시까지 동부 시간

* VOD/WebVTT에서 광고 삽입 후 자막이 동기화되지 않는 문제를 해결했습니다.

* 보안 업데이트.

### 버전 20.3.3

**시기:** 2020년 3월 26일 목요일, 오전 3시 ~ 오전 4시(동부 표준시)

* 이제 SSAI 4XX 및 5XX 응답이 CORS 관련 헤더를 올바르게 제공하므로 도메인 간 javascript webview 클라이언트가 오류 응답을 성공적으로 읽을 수 있습니다.

* 광고 서버에 전달할 때 IPv6 주소가 올바르게 URL로 인코딩되지 않은 X-Forwarded-For 헤더 문제를 해결했습니다.

* 특정 시나리오에서 EXT-X-MEDIA-SEQUENCE 번호가 잘못 증가하는 CMAF/demuxed 오디오 스트림 문제를 해결했습니다.

### 버전 20.3.2

**시기:** 2020년 3월 11일 수요일, 오전 5시 30분부터 오전 7시까지 동부 시간

* SCTE35 신호 처리 기능이 개선되었습니다.

* 유지 보수 업데이트.

### 버전 20.3.1

**시기:** 2020년 3월 5일 목요일, 오전 2:30 - 오전 4:30

* 성능 향상:

   * 마스터/미디어 m3u8 매니페스트에 대한 캐시 지원이 추가되었습니다. 이러한 매니페스트는 이제 캐시 제어: 공개 및 Max-Age 헤더에 응답하므로 비디오 시작 성능을 향상시키는 경우가 많습니다.

   * http를 통해 https 크리에이티브를 강제로 가져오도록 하여 비디오 시작 성능을 향상시킬 수 있는 지원이 추가되었습니다.

* 보안 및 유지 관리 수정 사항.

### 버전 20.2.1

**시기:** 2020년 2월 13일 목요일 오전 4:30 - 오전 5:30

* 언어/코덱/비트율을 기반으로 여러 오디오 전용 스트림을 포함하는 광고 에셋 결합에 대한 지원을 추가했습니다.
* 사소한 성능 향상 및 유지 보수 업데이트.

### 버전 20.1.3

**시기:** 2020년 1월 28일 화요일 오전 2시부터 오전 3시까지 동부 시간

* **nbc CueFormat에 대한 FER 지원 VMAP**

   다음과 같은 경우 FER 스트림의 큐를 FW 타임라인 재정의 매개 변수로 변환합니다. `ptcueformat=nbc` 가 사용되고 스트림은 매니페스트 내 큐와 구운 광고가 있는 VOD 스트림입니다.

* 서드파티 광고 공급자/CDN으로 전달하기 전에 HTTP 헤더의 사용자 에이전트 필드를 정리합니다.

* Auditude 및 기타 광고 공급자인 CDN으로 전송하기 전에 사용자 에이전트 HTTP 헤더에서 제어/인쇄 불가능한 문자(ASCII 코드 &lt; 32)를 필터링합니다. 잘못된 헤더에 대한 감사 광고 호출이 실패했습니다.

* NetStorage Groups에서 이전 V1 객체를 제거하여 객체 수를 Akamai의 안전 한도 이내로 유지합니다.

### 버전 20.1.2(핫픽스)

**시기:** 2020년 1월 20일 월요일 오전 2시~오전 3시(동부 표준시)

* 유지 보수 업데이트.

### 버전 20.1.1

**시기:** 2020년 1월 15일 수요일, 오전 4시 ~ 오전 5시(동부 표준시)

* Creative Repackaging Service는 이제 잘못된 형식의 크리에이티브를 자동으로 차단하여 더 빠른 광고 삽입 기능을 제공합니다.

* 유지 보수 업그레이드
