# Primetime Ad Insertion 20.12.1 릴리스 노트

Primetime Ad Insertion 20.12.1 릴리스 노트는 2020년 Primetime Ad Insertion의 새로운 기능 또는 변경된 기능, 해결된 문제 및 알려진 문제를 설명합니다.

## PTAI 20.12.1의 새로운 기능

**2020년 12월 8일** 화요일 오전 01:00에서 오전 04:00 동부 표준시

**변경 사항**

* 2020년 11월 30일 Primetime Ad Insertion에서 발생하는 간헐적인 클라이언트 연결(5xx) 문제를 해결하기 위한 핫픽스가 포함되어 있습니다.

**2020년** 11월 5일 목요일 오전 2시부터 오전 5시(동부 표준시)

**변경 사항**

* 유지 관리 업데이트.

#### 버전 201.10.2

* 유지 관리 업데이트.

**변경 사항**

* 유지 관리 업데이트.

#### 버전 20.9.3

**2020년 9월 30일** 수요일 오전 3:30~오전 6:30 동부 표준시

**변경 사항**

* Bootstrap API 매개 변수 `ptparallelstream`이(가) 추가되었습니다. 이를 통해 오디오 및 비디오 트랙의 광고가 일관되게 나타나도록 CMAF 요청 오디오 또는 비디오 스트림을 병렬로 요청하는 플레이어를 사용할 수 있습니다. 이 기능을 활성화하거나 비활성화하려면 매개 변수 값을 true로 설정합니다.

#### 버전 20.9.2

**2020년 9월 15일** 화요일 오전 3시 30분부터 오전 6시 30분까지 동부 표준시

**향상된 기능**

* `EXT-X-MARKER` 태그를 사용하여 비선형 광고 유형 포함에 대한 지원을 제공했습니다.
자세한 내용이나 이 기능을 활성화하려면 기술 지원 담당자에게 문의하십시오.

* 제공자가 응답하는 데 시간이 너무 오래 걸리는 경우 전체 광고 해상도 시간을 제한하기 위한 지원을 제공했습니다. 제한을 활성화하려면 부트스트랩 API 매개 변수 `ptadtimeout`을(를) 밀리초 단위로 설정합니다.

   >[!NOTE]
   >
   >이 시간 제한은 광고 크리에이티브 요청에는 적용되지 않고 광고 요청에만 적용됩니다.

#### 버전 20.9.1

**2020년 9월 1일** 화요일 오전 3시 30분부터 저녁 7시 30분까지 동부 표준시

**변경 사항**

* 때때로 DVR 창에서 EXT-X-MAP 토큰 또는 EXT-X-MAP 태그가 잘못 롤아웃되는 HLS/CMAF를 사용하는 고객에 대한 문제를 수정했습니다.

비디오 클라이언트에 대한 스트림 동기화를 밀리초 단위의 정확성이 요구되도록 개선했습니다. `#EXT-X-PROGRAM-DATE-TIME tags`에 대한 밀리 정밀도를 활성화하려면 Adobe 지원 센터에 문의하십시오.

#### 버전 20.6.1

**2020년 6월 2일** 화요일 오전 03:00에서 오전 05:00까지 동부 시간

**새로운 기능**

서버측 구성을 통해 다음과 같은 새로운 기능을 활성화하려면 Adobe 지원 센터에 문의하십시오.

* 매니페스트 조작:이제 HTTP와 HTTPS 간에 HLS 세그먼트 및 리소스 URL을 변형하여 백엔드 요청에서 TLS 핸드셰이크를 줄여 성능을 향상시킬 수 있습니다. 또한 광고/컨텐츠 조각을 동일한 CDN에 통합하는 데 사용할 수 있습니다.

* 긴 양식 VOD:긴 형식의 VOD 에셋을 사용하여 세션을 지속적으로 유지하기 위한 API를 개선했습니다.

**버그 수정**

* WebVTT 조각이 요청된 원래 프로토콜에 관계없이 http 프로토콜에서 항상 요청되던 문제를 수정했습니다.

* 광고에서 다시 콘텐츠로 전환할 때 EXT-X-DISCONTINUITY 태그가 재생 목록 맨 위에서 제거되었던 문제를 수정했습니다. 이 수정을 활성화하려면 Adobe 지원 센터에 문의하십시오.

* CRS 대시보드의 버그 수정.

#### 버전 20.3.4

* VOD/WebVTT에서 광고 삽입 후 자막이 동기화되지 않는 문제를 수정했습니다.

* 보안 업데이트.

#### 버전 20.3.3

**2020년** 3월 26일 목요일 오전 03:00에서 오전 04:00 동부 표준시

* 이제 SSAI 4XX 및 5XX 응답에서 CORS 관련 헤더를 올바르게 제공하여 크로스 도메인 javascript 웹뷰 클라이언트가 오류 응답을 성공적으로 읽을 수 있도록 합니다.

* 광고 서버로 전달될 때 IPv6 주소가 올바른 URL로 인코딩되지 않았던 X-Forwarded-For 헤더의 문제가 수정되었습니다.

* 특정 시나리오에서 EXT-X-MEDIA-SEQUENCE 번호가 잘못 증가되는 CMAF/demuxed 오디오 스트림 문제를 수정했습니다.

* 향상된 성능:

   * http를 통해 강제로 https 크리에이티브를 가져오는 데 대한 지원을 추가했으며, 이 경우 비디오 시작 성능도 향상시킬 수 있습니다.

* 보안 및 유지 관리 수정

#### 버전 20.2.1

**2020년 2** 월 13일 목요일 오전 04:30부터 오전 05:30까지 동부 시간

#### 버전 20.1.3

**2020년** 1월 28일 화요일 오전 2시부터 오전 3시까지 동부 표준시

* **NBC CueFormat에 대한 FER 지원을 통한 VMAP**

   `ptcueformat=nbc`이 사용되고 스트림이 인매니페스트 큐와 구워진 광고가 있는 VOD 스트림인 경우 FER 스트림의 큐를 FW 타임라인 재정의 매개 변수로 변환합니다.

* 제3자 광고 공급자/CDN으로 전달하기 전에 HTTP 헤더의 사용자-에이전트 필드를 정리합니다.

* Auditude 및 기타 광고 공급자, CDN으로 보내기 전에 사용자 에이전트 HTTP 헤더에서 제어/인쇄 불가능 문자(ASCII 코드 &lt; 32)를 필터링합니다. 이러한 잘못된 헤더에 대해 실패하기 위해 사용된 Auditude Ad-Call.

* NetStorage 그룹에서 이전 V1 객체를 제거하여 객체 수를 Akamai의 보호 제한 내에 유지합니다.
