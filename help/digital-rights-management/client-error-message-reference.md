# DRM 클라이언트 오류 메시지 참조 {#client-error-message-reference}

DRM 클라이언트 오류는 TVSDK 클라이언트측 오류의 하위 집합이며, DRM 관련 오류 코드는 3300~3399입니다.

## DRM 클라이언트 오류 {#drm-client-errors}

| 오류 코드 | 연상 기호 | 치료 |
|---|---|---|
| 3300 | 잘못된 바우처 | 배포자의 소프트웨어가 수행해야 하는 작업: <ul><li> Google Chrome을 사용 중이고 시크릿 모드에 있는 경우 Flash Player 버전이 11.6 미만이면 이 오류가 발생할 수 있습니다. 플레이어가 브라우저의 버전 번호를 확인하고 사용자에게 시크릿 모드를 종료하도록 조언하는 것이 좋습니다.</li><li> 라이선스를 다시 요청하십시오. 요청이 성공하면 로그인하거나 에스컬레이션하지 않아도 됩니다. 요청이 성공하지 못한 경우 오류의 원인이 된 콘텐츠를 기록합니다. `subErrorId` 줄 오류가 있는 경우 이 오류가 표시됩니다.</li></ul> 배포자가 수행해야 하는 작업: <ul><li> 버전 11.6 이하의 Flash을 사용하는 Chrome 이외의 구성에서 다시 시도가 성공하지 않으면 패키지 구성에서 오류가 발생했을 수 있습니다.</li><li>문제가 특정 콘텐츠에만 해당되는지 확인하고 다시 패키징합니다.</li></ul> |
| 3301 | 인증 실패 | 서버가 클라이언트를 인증하거나 인증하지 못했습니다.<ul><li>배포자의 소프트웨어는 사용자의 자격 증명을 다시 설정하거나 사용자가 콘텐츠에 대한 액세스 권한을 획득하도록 유도하는 데 필요한 모든 조치를 취해야 합니다.</li><li>배포자는 권한 부여 및 인증 메커니즘이 올바르게 작동하는지 확인해야 합니다. 배포자가 인증 또는 권한 부여 기능을 사용할 계획이 없는 경우 문제가 되는 콘텐츠의 정책에 인증이 필요한지 확인하고 정책/라이선스 불일치 진단 을 참조하십시오.</li></ul> 이 오류 코드에 대한 자세한 내용은 [DRM 오류 3301로 인해 및 해결 방법](https://forums.adobe.com/thread/1277149). |
| 3302 | SSL 필요 | Primetime DRM 4.0 이상의 경우, 이 오류는 원격 키 URL이 HTTPS를 구성표로 사용하지 않을 때 iOS에서 발생합니다. HTTPS는 필수입니다. <ul><li>배포자가 Primetime DRM 버전 4 이하 또는 버전 4 이상을 사용하지만 플랫폼이 iOS이 아닌 경우 배포자의 소프트웨어가 오류를 기록해야 합니다. 이 오류는 iOS에서만 발생합니다.</li><li>배포자의 소프트웨어가 Primetime DRM 버전 4 이상이고 플랫폼이 iOS인 경우 배포자는 사용 중인 원격 키 서버 URL을 HTTPS로 변경해야 합니다. HTTP만 사용하는 경우 배포자가 HTTPS 서버를 설정해야 할 수 있습니다. 그렇지 않으면 배포자는 로그된 정보를 Adobe에 제출하고 문제를 에스컬레이션해야 합니다.</li></ul> |
| 3303 | ContentExpired | 보고 있는 콘텐츠는 콘텐츠 공급자가 설정한 규칙에 따라 만료되었습니다.  `subErrorId` 클라이언트별 오류 또는 줄 오류가 포함되어 있습니다. <ul><li>배포자의 소프트웨어는 만료되지 않은 새 라이센스를 사용할 수 있는지 확인하기 위해 서버에서 라이센스를 한 번 다시 받아야 합니다. 사용할 수 있는 라이선스가 없거나 라이선스가 만료된 경우 사용자에게 새 라이선스를 취득하도록 허용하거나 콘텐츠를 볼 수 없음을 사용자에게 알립니다.</li></ul> 콘텐츠가 만료/종료 날짜가 경과된 정책으로 패키지화되고 라이선스 서버가 PolicyEvaluationException을 보고하는 경우, 정책 종료 날짜가 경과되었다고 명시합니다(서버 오류 코드 303). 서버의 로그 파일을 확인하여 확인합니다.<br>가능하면 포장 중에 사용한 정책은 유효기간이 지났는지 여부를 고객이 확인해야 한다. Java 명령줄 도구는 다음과 같습니다. <ul><li>java -jar libs/AdobePolicyManager.jar</li><li> 세부 사항 demo.pol</li><li>배포자는 라이선스 만료 날짜가 의도한 대로 구성되어 있는지 확인해야 합니다.</li></ul> 이 오류 코드에 대한 자세한 내용은 [라이브 스트림을 사용하여 AMS/FMS를 사용하는 3303(콘텐츠가 만료됨)?](https://forums.adobe.com/thread/1300813) |
| 3304 | AuthorizationFailed | 현재 사용자는 콘텐츠를 볼 수 있는 권한이 없습니다. 다른 사용자로 로그인합니다.<br>이 오류 코드에 대한 자세한 내용은 오류 코드 3301을 참조하십시오. |
| 3305 | ServerConnectionFailed | 네트워크 지연 또는 클라이언트가 오프라인 상태이므로 라이선스 또는 도메인 서버에 대한 연결 시간이 초과되었습니다. 일반적으로 subErrorId에는 HTTP 반환 코드가 포함됩니다.<ul><li>배포자의 소프트웨어가 정상 작동이 확인된 서버에 네트워크 연결을 시도해야 합니다. 실패할 경우 네트워크에 다시 연결하라는 메시지를 표시합니다. 시도가 성공하면 기록합니다.</li><li>배포자는 사용 중인 모든 라이선스 및 도메인 서버가 온라인 상태이고 클라이언트의 네트워크에서 볼 수 있는지 확인해야 합니다.</li></ul>이 오류 코드에 대한 자세한 내용은 [DRM 3305 ServerConnectionFailed 원인 및 해결 방법](https://forums.adobe.com/thread/1284947.) |
| 3306 | ClientUpdateRequired | 현재 클라이언트는 요청된 작업을 완료할 수 없지만 업데이트된 클라이언트는 요청을 완료할 수 있습니다.<br>여기에는 다음과 같은 몇 가지 원인이 있을 수 있습니다.<ul><li>이 클라이언트에서 사용할 수 없는 공유 도메인이 사용되었습니다. 이는 재생이 Chrome에서 작동하지만 다른 브라우저에서는 작동하지 않는 경우이거나 그 반대일 수 있습니다.<br>**참고:** Chrome은 다른 브라우저에서 사용하는 것과 다른 PHDS/PHLS 키를 사용합니다.</li><li>5.0 이전 버전의 iOS에서 실행할 때 애플리케이션에서 여러 DRMSesections를 추가하려고 합니다.</li><li>버전 2만 지원되는 경우 메타데이터의 버전이 3 이상입니다.</li><li>배포자의 소프트웨어는 사용자에게 경고하고 작업을 중단해야 합니다. 소프트웨어에서 업그레이드를 사용할 수 있는지 여부를 확인하는 방법이 있는 경우 플랫폼에 적합한 방식으로 해당 업그레이드를 사용자에게 안내합니다.</li><li>공유 도메인으로 인해 문제가 발생하면 배포자는 Adobe에 업데이트된 런타임 또는 라이브러리를 확인해야 합니다. Flash 런타임의 경우 배포자가 애플리케이션에서 직접 업그레이드를 강제할 수 있습니다. 라이브러리의 경우 배포자는 업데이트된 라이브러리를 가져와 애플리케이션을 다시 빌드하고 사용자에게 배포해야 합니다.</li></ul>여러 DRMSessions로 인해 문제가 발생하는 경우 배포자는 여러 DRMSessions를 추가하기 전에 iOS 버전 번호를 확인하기 위해 애플리케이션을 업데이트해야 합니다. 또는 애플리케이션 배포를 iOS 버전 5 이상으로 제한할 수 있습니다.<br>메타데이터 버전이 버전 2보다 높기 때문에 문제가 발생하는 경우 메타데이터가 손상되었을 수 있습니다. 메타데이터를 다시 빌드하고 결과를 살펴볼 수 있습니다. 문제가 계속 표시되면 문제를 기록하고 Adobe으로 에스컬레이션합니다.<br>이 오류 코드에 대한 자세한 내용은 [3306 DRMErrorEvent 오류 코드를 해결하는 방법](https://forums.adobe.com/thread/1266675). |
| 3307 | 내부 오류 | 이는 일반적으로 Primetime DRM 코드의 버그를 나타내며 아래와 같이 알려진 버그가 없는 한 예기치 않습니다.subErrorId에는 클라이언트별 오류 또는 줄 오류가 포함되어 있습니다.<ul><li>브라우저가 Windows의 Chrome이고 Flash 버전이 11.6(SWF 버전 19 이상)인 경우 배포자의 소프트웨어는 사용자가 infovar에서 Deny를 눌렀다고 가정하고 동일한 버전을 3368로 처리해야 합니다.</li><li>브라우저가 Chrome이 아니거나 Flash 버전이 11.6이 아닌 경우 3307이 발생하면 배포자가 Adobe으로 에스컬레이션해야 합니다.</li></ul>**참고:** 3307:1107296344 (FailedToGetBrokerHandle)은 Chrome 브라우저 버전 24-28에서 발생할 수 있습니다. |
| 3308 | 잘못된 라이센스 키 | 이 오류는 사용 중인 라이선스에 콘텐츠를 해독할 잘못된 키가 있을 때마다 발생합니다. subErrorId에는 클라이언트별 오류나 줄 오류가 포함되어 있습니다.<br>이 오류를 생성하는 방법에는 두 가지가 있습니다.<ul><li>고객이 라이센스를 생성하기 위한 표준 Adobe 도구(예: 라이센스 서버 Java 프레임워크)를 수정했습니다. 이 경우, 라이센스에 콘텐츠에 해당하지 않을 수 있는 잘못된 키가 포함되어 있습니다.</li><li>고객이 동일한 라이선스 ID로 여러 라이선스를 발급했습니다. 이 경우 클라이언트에서 사용할 수 있는 여러 라이센스가 컨텐츠 메타데이터와 일치하며 Primetime DRM 코드가 잘못된 라이센스를 선택하여 사용합니다.</li><li>배포자의 소프트웨어는 서버에서 라이선스를 다시 획득해야 합니다.</li><li>사용 가능한 라이선스가 없거나 라이선스가 만료된 경우 사용자에게 새 라이선스를 취득할 수 있는 워크플로우를 제공하거나 콘텐츠를 볼 수 없음을 사용자에게 알리고 문제를 기록합니다.</li><li> 도메인 바인딩된 콘텐츠(AIR용)인 경우 사용자가 도메인에 가입할 수 있는 방법을 제공하십시오.</li><li>배포자는 다음과 같아야 합니다.</li><li> Primetime DRM 라이센스 서버의 라이센스 발급 부분을 사용자 지정하지 않았는지 확인합니다.</li><li>모든 라이센스에 대해 고유 라이센스 ID를 발급하고 있는지 확인합니다.</li><li>Adobe으로 문제를 에스컬레이션합니다.</li></ul> |
| 3309 | 손상된 추가 헤더 | 이 문제는 헤더가 65536바이트보다 큰 경우에 발생합니다.<ul><li>배포자의 소프트웨어는 오류가 발생한 콘텐츠 부분을 기록해야 합니다.</li><li>디스트리뷰터는 특정 함량으로 오류를 재현할 수 있는지 확인해야 합니다. 깨진 콘텐츠를 다시 패키징합니다.</li></ul> |
| 3310 | AppIDMismatch | 응용 프로그램이 허용 목록에 추가되지 않았습니다. Android, iOS 또는 Flash SWF.<br>subErrorId: 1000942; 보호된 스트림을 재생하는 동안 오류가 발생했습니다. FAXS 오류.<br>또한 클라이언트가 pubID(앱 게시자 ID)에 대한 빈 문자열을 보고하고 있을 수 있습니다.<br>**Android:** Android 애플리케이션이 사용 중인 애플리케이션과 일치하지 않습니다. Android에서 디버그 키 저장소의 디렉터리는 릴리스 키 저장소의 디렉터리와 다른 경우가 많다는 점을 기억하십시오.<br>**iOS:** 다음을 참조하십시오. [iOS 애플리케이션 허용 목록](/help/programming/tvsdk-3x-ios-prog/ios-3x-drm-content-security/ios-3x-allowlist-your-ios-application.md) tvsdk iOS 안내서의 설명서입니다. |
| 3312 | LicenseIntegrity | 서버에서 라이선스를 다시 다운로드합니다. |
| 3313 | WriteMicrosafeFailed | 이 문제는 시스템이 파일 시스템에 쓸 수 없을 때 발생합니다. subError클라이언트별 오류 또는 줄 오류를 포함합니다.<br>Microsoft Windows에서 암호화된 콘텐츠에 licenseID 또는 policyID가 너무 긴 경우 Active X 또는 NPAPI 플러그 플래시 플레이어에서 오류 3313이 발생할 수 있습니다. 이는 Windows의 최대 경로 길이 때문입니다. (후추 플러그인은 이 문제가 없습니다.)<ul><li>배포자의 소프트웨어는 사용자에게 사용자 디렉토리가 잠겨 있지 않은지 또는 꽉 찼거나 잠긴 볼륨에서 잠겨있는지 확인하라는 메시지를 표시해야 합니다.</li><li>배포자가 Flash이 아닌 AIR을 사용하는 경우 경로 길이 제한으로 인해 문제가 발생할 수 있습니다. 배포자는 AIR 애플리케이션의 이름을 합리적으로 단축해야 합니다.</li></ul> |
