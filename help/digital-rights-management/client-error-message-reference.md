# DRM 클라이언트 오류 메시지 참조 {#client-error-message-reference}

DRM 클라이언트 오류는 TVSDK 클라이언트측 오류의 하위 집합이며 DRM 관련 오류 코드는 3300에서 3399까지입니다.

## DRM 클라이언트 오류 {#drm-client-errors}

| 오류 코드 | 니모닉 | Remedy |
|---|---|---|
| 3300 | 잘못된 할인권 | 배포자의 소프트웨어가 해야 할 일: <ul><li> Google Chrome을 사용하고 있으며 Incognito 모드에 있고 Flash Player 버전이 11.6 미만인 경우 이 오류가 발생할 수 있습니다. 플레이어가 브라우저의 버전 번호를 확인하고 Incognito 모드를 종료하도록 권장됩니다.</li><li> 라이선스를 다시 요청합니다. 요청이 성공하면 로그인하거나 문제 제기를 할 필요가 없습니다. 요청이 실패할 경우 오류를 일으킨 컨텐츠를 기록합니다. `subErrorId` 에는 라인 오류가 있습니다(있는 경우).</li></ul> 배포자가 해야 할 작업: <ul><li> Flash가 11.6보다 작은 크롬 이외의 구성에서 재시도가 실패할 경우 패키징에 오류가 발생했을 수 있습니다.</li><li>특정 컨텐츠와 재패키징에 문제가 있는지 확인합니다.</li></ul> |
| 3301 | 인증 실패 | 서버가 클라이언트를 인증하거나 인증하지 못했습니다.<ul><li>배포자의 소프트웨어는 사용자의 자격 증명을 재설정하거나 사용자가 컨텐츠에 액세스할 수 있도록 안내하는 데 필요한 조치를 취해야 합니다.</li><li>배포자는 인증 및 인증 메커니즘이 제대로 작동하는지 확인해야 합니다. 배포자가 인증 또는 인증 기능을 사용할 계획이 없는 경우, 문제가 있는 컨텐츠의 정책에 인증이 필요한지 여부를 확인하고 정책/라이센스 불일치 진단을 참조하십시오.</li></ul> 이 오류 코드에 대한 자세한 내용은 [DRM 오류 3301 원인 및 해상도를 참조하십시오](https://forums.adobe.com/thread/1277149). |
| 3302 | RequireSSL | Primetime DRM 4.0 이상의 경우 원격 키 URL이 구성표로 HTTPS를 사용하지 않으면 iOS에서 이 오류가 발생합니다. HTTPS가 필요합니다. <ul><li>배포자가 Primetime DRM 버전 4 이전 버전 또는 버전 4 이상을 사용하고 있지만 플랫폼이 iOS가 아닌 경우, 배포자의 소프트웨어는 오류를 기록해야 합니다. 이 오류는 iOS에서만 발생합니다.</li><li>배포자의 소프트웨어가 Primetime DRM 버전 4 이상이고 플랫폼이 iOS인 경우 배포자는 사용 중인 원격 키 서버 URL을 HTTPS로 변경해야 합니다. HTTP만 사용하는 경우 배포자는 HTTPS 서버를 설정해야 할 수 있습니다. 그렇지 않은 경우, 배포자는 기록된 정보를 Adobe에 제출하고 문제를 제기해야 합니다.</li></ul> |
| 3303 | 컨텐츠 만료됨 | 현재 보고 있는 콘텐트가 콘텐트 공급자가 설정한 규칙에 따라 만료되었습니다.  `subErrorId` 에는 클라이언트별 오류 또는 라인 오류가 있습니다. <ul><li>배포자의 소프트웨어는 사용권이 만료되지 않은 새로운 라이센스를 사용할 수 있는지 여부를 결정하기 위해 서버로부터 라이센스를 다시 획득하려고 시도해야 합니다. 사용 가능한 라이센스가 없거나 라이센스가 만료된 경우 사용자가 새 라이센스를 획득하도록 허용하거나 컨텐츠를 볼 수 없다는 사실을 사용자에게 알립니다.</li></ul> 컨텐츠가 만료/종료 날짜가 지난 정책으로 패키지되었으며 라이선스 서버에서 PolicyEvaluationException을 보고하는 경우 정책 종료 날짜가 만료되었음을 나타냅니다(서버 오류 코드 303). 서버의 로그 파일을 확인하여 확인합니다.<br>가능한 경우 패키지 과정에서 사용한 정책을 확인하여 만료되었는지 확인해야 합니다. Java 명령줄 도구는 다음과 같습니다. <ul><li>java -jar libs/AdobePolicyManager.jar</li><li> detail demo.pol</li><li>배포자는 라이센스 만료 날짜가 의도한 대로 구성되어 있는지 확인해야 합니다.</li></ul> 이 오류 코드에 대한 자세한 내용은 라이브 스트림을 사용하는 AMS/FMS와 [함께 3303(콘텐츠 만료)을 참조하십시오.](https://forums.adobe.com/thread/1300813) |
| 3304 | 권한 부여 실패 | 현재 사용자는 콘텐트를 볼 수 있는 권한이 없습니다. 다른 사용자로 로그인합니다.<br>이 오류 코드에 대한 자세한 내용은 오류 코드 3301을 참조하십시오. |
| 3305 | ServerConnectionFailed | 네트워크 지연 또는 클라이언트가 오프라인 상태이므로 라이센스 또는 도메인 서버에 대한 연결이 시간 초과되었습니다. NormalSubErrorId에는 HTTP 반환 코드가 포함되어 있습니다.<ul><li>배포자의 소프트웨어는 정상 작동이 확인된 서버에 대한 네트워크 연결을 시도해야 합니다. 시도가 실패하면 네트워크에 다시 연결하라는 메시지를 표시합니다. 시도가 성공하면 이를 기록합니다.</li><li>배포자는 사용 중인 라이센스 및 도메인 서버가 온라인 상태이며 클라이언트의 네트워크에서 볼 수 있는지 확인해야 합니다.</li></ul>이 오류 코드에 대한 자세한 내용은 [DRM 3305 ServerConnectionFailed 원인 및 해상도를 참조하십시오.](https://forums.adobe.com/thread/1284947.) |
| 3306 | ClientUpdateRequired | 현재 클라이언트가 요청한 작업을 완료할 수 없지만 업데이트된 클라이언트가 요청을 완료할 수 있습니다.<br>여기에는 몇 가지 원인이 있을 수 있습니다.<ul><li>이 클라이언트에서 사용할 수 없는 공유 도메인이 사용되었습니다. Chrome에서 재생되는 경우도 있지만 다른 브라우저에서는 그렇지 않고 그 반대의 경우도 마찬가지입니다.<br>**참고:**크롬은 다른 브라우저에서 사용하는 것과 다른 PHDS/PHLS 키를 사용합니다.</li><li>5.0 이전의 iOS 버전에서 실행할 때 응용 프로그램에서 여러 DRMSessions를 추가하려고 합니다.</li><li>버전 2만 지원되는 경우 메타데이터의 버전이 3 이상인 경우</li><li>배포자의 소프트웨어는 사용자에게 경고해서 작업을 중단합니다. 소프트웨어의 업그레이드 사용 가능 여부를 결정할 수 있는 방법이 있는 경우, 사용자에게 해당 플랫폼에 맞는 방식으로 업그레이드를 지시하십시오.</li><li>공유 도메인으로 인해 문제가 발생하는 경우, 배포자는 Adobe에 업데이트된 런타임 또는 라이브러리를 확인해야 합니다. Flash 런타임의 경우, 배포자는 애플리케이션에서 직접 업그레이드를 강제할 수 있습니다. 라이브러리의 경우 배포자는 업데이트된 라이브러리를 구하고 애플리케이션을 다시 구축하여 사용자에게 배포해야 합니다.</li></ul>여러 DRMSessions로 인해 문제가 발생하는 경우, 배포자는 여러 DRMS를 추가하기 전에 iOS 버전 번호를 확인하기 위해 응용 프로그램을 업데이트해야 합니다. 또는 응용 프로그램의 iOS 버전 5 이상으로 배포를 제한할 수 있습니다.<br>메타데이터 버전이 버전 2보다 높으므로 문제가 발생하는 경우 메타데이터가 손상되었을 수 있습니다. 메타데이터를 다시 작성하고 결과를 볼 수 있습니다. 문제가 계속 표시되는 경우 해당 문제를 로그인하여 Adobe로 문제 제기하십시오.<br>이 오류 코드에 대한 자세한 내용은 3306 DRMErrorEvent 오류 코드 [를 해결하는 방법을 참조하십시오](https://forums.adobe.com/thread/1266675). |
| 3307 | 내부 실패 | Primetime DRM 코드의 버그를 일반적으로 나타내며 아래에 알려진 버그가 없는 경우 예기치 않은 문제가 됩니다.subErrorId에는 클라이언트별 오류나 라인 오류가 있습니다.<ul><li>브라우저가 Windows에서 Chrome이고 Flash 버전이 11.6(SWF 버전 19 이상)인 경우, 배포자의 소프트웨어는 사용자가 소프트웨어에 대해 Deny를 눌렀다고 가정하고 사용자가 3368과 동일하게 취급해야 합니다.</li><li>브라우저가 Chrome이 아니거나 Flash 버전이 11.6이 아닌 경우, 3307이 발생하는 경우, 배포자는 Adobe로 문제를 제기해야 합니다.</li></ul>**참고:** 3307:1107296344(FailedToGetBrokerHandle)는 Chrome 브라우저 버전 24-28에서 발생할 수 있습니다. |
